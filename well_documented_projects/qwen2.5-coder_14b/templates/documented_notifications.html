It includes a list of existing notifications, each with options to delete them, as well as a form to create new notifications. #### 2. Key Functions/Methods and Their Responsibilities - **Display Notifications**: The template renders a list of notifications passed from the backend. Each notification includes its message and associated role. - **Delete Notification**: Users can submit a form to delete a specific notification by clicking the "Delete" button next to each notification. - **Create New Notification**: A form is provided for users to input a new notification message and select a role, which are then submitted to the server. #### 3. Inputs/Outputs/Side Effects - **Inputs**: - `notifications`: A list of notification objects passed from the backend containing `message` and `role`. - **Outputs**: - HTML content displaying notifications and forms for creating and deleting them. - **Side Effects**: - The user can delete or create notifications, which affects the state on the server. #### 4. Design Patterns, Dependencies - **Design Patterns**: The template follows a simple client-side rendering pattern using Django templates. - **Dependencies**: - Django framework for templating and handling form submissions. - Static CSS file for styling (`calendar.css`). #### 5. Cohesion and Coupling - **Cohesion**: High, as the HTML elements are closely related to displaying and managing notifications. - **Coupling**: Low, as the template is decoupled from the backend logic, relying solely on data passed in through context variables.
