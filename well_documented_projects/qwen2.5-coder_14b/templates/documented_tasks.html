It includes features for displaying tasks, marking them as complete, deleting tasks, and adding new tasks. #### Key Functions/Methods and Their Responsibilities - **Navigation Links**: Provide access to the dashboard and calendar. - **Message Display**: Show messages (e.g., success or error) that persist for 5 seconds before disappearing. - **Task List Section**: Displays a table of tasks with details such as title, description, assigned user, course, section, due date, status, and actions (mark complete/delete). - **Add Task Section**: Allows users to create new tasks by filling out a form with required fields like task name, description, due date, and assignment. #### Inputs/Outputs/Side Effects - **Inputs**: - User interactions via forms for marking tasks as complete, deleting tasks, or adding new tasks. - Data passed from the server such as `tasks`, `messages`, `users`, `courses`, and `sections`. - **Outputs**: - Displayed HTML content on the web page based on the provided data. - User actions processed by the backend (e.g., task completion, deletion). #### Design Patterns, Dependencies - **Template Inheritance**: Uses Django template tags for dynamic content rendering. - **CSS**: Relies on an external CSS file (`calendar.css`) for styling. #### Cohesion and Coupling - **Cohesion**: High cohesion within each section of the page. Each section has a clear purpose (task list, message display, add task). - **Coupling**: Low coupling between different sections. They are independent but interact with each other through user inputs and server-provided data.
